# Carbon Commute Tracker

Java OOP project for tracking daily commute trips and calculating CO2 emissions.

Authors: Noor Ul Eman and Hafsa Tahir  
Language: Java 11+

## What This Project Does

The app lets a user:

1. Register or select a user.
2. Add a commute trip.
3. Choose a transport mode, such as car, bus, bicycle, or walking.
4. Enter distance in kilometers.
5. Calculate CO2 emission.
6. View history and reports.
7. Save/load data from a file.

Basic formula:

```text
CO2 emission = distance in km * emission factor
```

Example:

```text
Car emission factor = 0.21 kg/km
Distance = 10 km
CO2 = 10 * 0.21 = 2.10 kg
```

## Main OOP Concepts

### Encapsulation

Fields are private, and other classes use methods/getters.

Example:

```java
private String name;
public String getName() {
    return name;
}

````

### Inheritance

Transport classes extend `TransportMode`.

Example:

```java
public class Car extends TransportMode
```

### Abstraction

`TransportMode` is abstract. It says every transport mode must know how to calculate emissions.

```java
public abstract double calculateEmission(double distanceKm);
```

### Polymorphism

The program can use one type, `TransportMode`, but the real object can be `Car`, `Bus`, `Walking`, etc.

```java
TransportMode mode = new Car();
mode.calculateEmission(10);
```

### Composition

One class contains objects of another class.

Examples:

```text
User has many CommuteRecords
CommuteRecord has one TransportMode
CarbonCommuteSystem has many Users
```

##  Conclusion:

This project is a Carbon Commute Tracker. A user can register, add daily commute records, select a transport mode, and enter distance. The program calculates CO2 emission using the formula distance multiplied by emission factor. The project demonstrates OOP concepts: encapsulation through private fields and getters, inheritance through transport classes extending `TransportMode`, abstraction through the abstract `calculateEmission` method, polymorphism because each transport type calculates emission differently, and composition because users contain commute records. Data is saved in a text file using `FileHandler`, and reports are generated by `EmissionReport`.
